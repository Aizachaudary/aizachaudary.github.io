Advanced DBMS Platform

A real-time streaming pipeline with a natural-language-to-SQL interface, built for an Advanced Database Management Systems course — but engineered like a production system, not a course demo.

CourseAdvanced DBMS
RoleSolo build
EnvironmentWindows 11 + Docker
StatusDeployed & demoed

Overview

The brief was to design a database system that goes beyond static queries. I built a pipeline that ingests data in real time, indexes it for semantic search, and lets a user ask questions in plain English instead of writing SQL by hand — the system translates the question into a query, runs it, and returns results.

Architecture

Five components, each handling one stage of the pipeline:

1# pipeline.yml
2ingest: Apache Kafka # streams incoming data
3process: Apache Spark # transforms + batches records
4store: PostgreSQL # source of truth
5search: FAISS # vector index for semantic lookup
6interface: FastAPI + Groq API # NL question → SQL query

A user's natural-language question is embedded with sentence-transformers, matched against context in FAISS, and passed to an LLM (via the Groq API) that generates the corresponding SQL — which then runs against PostgreSQL and returns live results through FastAPI.

Challenges

Windows + Docker environment conflicts
Kafka and Spark are built with Linux-first assumptions. Getting both running reliably in Docker containers on Windows 11 meant resolving networking, volume-mount, and port-binding issues one by one rather than following a single tutorial.
Keeping the streaming layer in sync with the vector index
New data arriving through Kafka needed to be reflected in the FAISS index without re-embedding everything from scratch each time, so the pipeline had to handle incremental updates correctly.
Reliable NL-to-SQL translation
Getting an LLM to consistently produce valid, safe SQL from an open-ended question — rather than syntactically broken or overly literal queries — required careful prompt design and validation before execution.
