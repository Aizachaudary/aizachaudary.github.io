HairGenie

An AI-powered haircare recommendation platform built for the Advanced Database Management Systems course, with a mandatory live-deployment requirement.

CourseAdvanced DBMS
RoleSolo build
FrontendNext.js
StatusIn progress

Overview

HairGenie recommends haircare products and routines personalized to a user's hair type, concerns, and history. The course requirement wasn't just "use a database" — it was to design a system that genuinely needed more than one kind of database, and to ship it as a live, working deployment rather than a local demo.

Architecture

Each database in the stack does a specific job rather than being used interchangeably:

1// architecture.json
2frontend: "Next.js"
3auth_+_db: "Supabase (Postgres-backed)"
4structured: "PostgreSQL — users, products, orders"
5flexible: "MongoDB — varied product attributes, reviews"
6cache: "Redis — fast recommendation lookups"

PostgreSQL holds the relational core (users, orders, product catalog), MongoDB stores semi-structured data like varied product specs and free-form reviews, and Redis caches frequently requested recommendations so the app doesn't recompute them on every request.

Challenges

Justifying a genuine multi-database design
It's easy to bolt on extra databases without a real reason. The harder part was designing data flows where each database's strengths (relational integrity, schema flexibility, speed) actually mattered to the product.
Mandatory live deployment
The course required a publicly reachable deployment, not a local build — which meant handling environment variables, database connection strings, and hosting configuration correctly across three separate data stores.
Personalization logic without an ML pipeline
Building meaningful, personalized recommendations using rule-based logic and structured queries — rather than reaching for an ML model when the course scope called for database design fundamentals.

Status

Several pages are still being finished. The core database architecture, auth flow, and recommendation logic are in place; remaining work is primarily on the frontend pages and final deployment polish.
