How I work

From coursework to something real

Every project follows the same path — understand the real constraint, design around it, then make sure it actually runs somewhere other than my laptop.

01
Understand the real constraint
Whether it's "use only foundational libraries" or "must support real-time data" — I design around the actual limitation, not a simplified version of it.
02
Build the full pipeline
Data layer, logic layer, interface — built and connected end to end, so the system works as a whole, not as disconnected parts.
03
Ship it, not just submit it
Where the project allows it, I deploy — a live URL, an APK, a working demo — because that's the difference between a grade and a portfolio piece.
